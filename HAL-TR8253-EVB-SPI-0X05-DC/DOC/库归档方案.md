# 库归档方案

SSC 协议栈 + HAL 驱动 → 编译为 `.lib`，永久封存。
业务工程师只动上层源码文件。

---

## 一、进库 (永久不变，只读)

### SSC 协议栈核心 — 12 个文件，SSC Tool 再怎么配置也不变

| 文件 | 说明 |
|------|------|
| `SSC/ecatslv.c` + `.h` | EtherCAT 状态机 |
| `SSC/ecatappl.c` + `.h` | PDO 引擎 + MainLoop |
| `SSC/ecatcoe.c` + `.h` | CoE 协议 |
| `SSC/mailbox.c` + `.h` | 邮箱收发队列 |
| `SSC/sdoserv.c` + `.h` | SDO 服务 |
| `SSC/coeappl.c` + `.h` | 通用对象字典初始化 + 0x1018 标识 |
| `SSC/objdef.c` + `.h` | 标准对象定义 |
| `SSC/applInterface.h` | SSC 对外接口声明 |
| `SSC/esc.h` | ESC 寄存器地址宏 |

### 平台移植

| 文件 | 说明 |
|------|------|
| `SSC/TR8253Evb.c` + `.h` | SPI 驱动 + GPIO 中断 + 定时器 |
| `SSC/el9800hw.c` + `.h` | 硬件抽象 |

### HAL 驱动 + 系统

| 文件 | 说明 |
|------|------|
| `Drivers/STM32F4xx_HAL_Driver/Src/*.c` | HAL 库 |
| `Core/Src/system_stm32f4xx.c` | 系统初始化 (固定) |

---

## 二、不进库 (独立源码，根据 PDO 配置变化)

| 文件 | 说明 | 什么时候改 |
|------|------|-----------|
| `SSC/SSC-DeviceObjects.h` | PDO 对象定义 + ApplicationObjDic | SSC Tool 重新生成后覆盖 |
| `SSC/SSC-Device.c` | APPL_Application / APPL_InputOutputMapping | SSC Tool 重新生成后覆盖 |
| `SSC/SSC-Device.h` | 设备声明 + 引入 ecat_cb.h | 只需补 1 行 `#include "ecat_cb.h"` |
| `SSC/ecat_def.h` | VENDOR_ID / MAX_PD_* 等全局宏 | SSC Tool 重新生成后覆盖 |

---

## 三、库编译条件

Keil → C/C++ → Misc Controls，对库里的 `.c` 文件加上：

```
--cpreinclude ecat_cb.h
```

> 库里的 `SSC-Device.c` 引用 `g_pfnPeriodicTask` 等 4 个函数指针，
> 这行让编译器在编译库之前自动注入 `ecat_cb.h`，SSC 源码一行不改。

## 文件树分类

```
项目/
├─ [库] SSC/                     ← 全部 .c + .h → ssc_hal.lib
├─ [库] Drivers/STM32F4xx_*/    ← 全部 → ssc_hal.lib
├─ [库] Core/Src/system_stm32f4xx.c
│
├─ [源] Core/Inc/ecat_cb.h       ← 函数指针 (库和源共用)
├─ [源] Core/Inc/usart.h
├─ [源] Core/Inc/stm32f4xx_*.h
├─ [源] Core/Inc/main.h
│
├─ [源] Core/Src/main.c
├─ [源] Core/Src/gpio.c
├─ [源] Core/Src/spi.c
├─ [源] Core/Src/tim.c
├─ [源] Core/Src/usart.c
├─ [源] Core/Src/stm32f4xx_it.c
├─ [源] Core/Src/stm32f4xx_hal_msp.c
│
├─ [源] ECAT/ecat_api.h + .c     ← 对外接口层
├─ [源] HARDWARE/Application.h + .c
├─ [源] HARDWARE/App_Flash.h + .c
│
└─ [关键] Keil 编译选项:
         --cpreinclude ecat_cb.h
```
