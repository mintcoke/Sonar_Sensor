# STM32F407 固件架构总览

> 只关注 MCU 内部运行，不涉及与 ESC/主站的通信细节。

---

## 一、上电到就绪

```
上电 → 复位向量 → SystemInit (汇编) → main() @ main.c
  │
  ├─ HAL_Init()                      ← SysTick 1ms 心跳
  ├─ SystemClock_Config()            ← HSE 8MHz → PLL → 168MHz, APB1=42MHz, APB2=84MHz
  ├─ MX_GPIO_Init()                  ← 26引脚方向/模式 + EXTI 中断线
  ├─ MX_SPI1_Init()                  ← SPI1 时钟+GPIO复用 (Mode 3, 8bit)
  ├─ MX_TIM3_Init()                  ← TIM3 1ms 时基
  │
  ├─ ECAT_RegisterPeriodicTask( App_Periodic_Task )      ← 存函数指针
  ├─ ECAT_RegisterSafeOutput( App_Safe_Output_State )    ← 存函数指针
  ├─ App_Hardware_Init()                                  ← 用户外设初始化 (空壳)
  │
  └─ ECAT_Stack_Init()
       ├─ 绑定内部桥接函数到全局指针
       ├─ 初始化 SPI 通信 (重试3次, 间隔100ms)
       ├─ App_Load_Params_From_Flash()  ← 从 Flash 恢复持久化参数到 RAM
       └─ 协议栈初始化 → ENABLE_GLOBAL_INT → 中断开始生效
```

## 二、稳态: while(1) 无限循环

```
while (1) {
    ECAT_Stack_MainLoop()
}
```

每次循环做三件事:

```
ECAT_Stack_MainLoop() @ ecat_api.c
  │
  ├─ ① 协议栈轮询
  │     └─ 内部会按需回调:
  │          ├─ App_Periodic_Task()      ← 你的业务逻辑 (读DI, 写DO, 处理传感器...)
  │          └─ App_Safe_Output_State()  ← 通信故障时触发
  │
  ├─ ② 输出看门狗 (100ms)
  │     通信正常 → 清零计数器
  │     通信中断超100ms → 调用 App_Safe_Output_State() → DO_WriteAll(0)
  │
  └─ ③ Flash 自动保存
        g_bConfigDirty == 1 → App_Save_Params_To_Flash() → g_bConfigDirty = 0
```

### 你的业务逻辑挂载点: App_Periodic_Task()

```
App_Periodic_Task() @ Application.c
  │
  ├─ 输入消抖: 8路DI, 垂直计数法, 连续3次一致才确认
  │    └─ DI_Read(0..7) → 消抖状态机 → 写入 ECAT_DI_VAR.DIx
  │
  ├─ 读输出: 从 ECAT_DO_VAR.DOx 获取主站指令
  │
  └─ 驱动输出: DO_Write(0..15, val) → BITBAND 单周期写 GPIO
```

### 安全输出挂载点: App_Safe_Output_State()

```
App_Safe_Output_State() @ Application.c
  └─ DO_WriteAll(0)   ← 所有输出清零
```

## 三、中断 (与 while(1) 并发)

两个外部中断随时打断主循环:

```
HAL_GPIO_EXTI_Callback(GPIO_Pin) @ TR8253Evb.c
  │
  ├─ GPIO_Pin == PA2 (ESC_INT)
  │     └─ PDI_Isr()  → 读 ESC 事件寄存器 → 标记 "有新数据/状态变化"
  │                     → 下一次 ① 协议栈轮询时处理
  │
  └─ GPIO_Pin == PA0 (ESC_SYNC0)
        └─ Sync0_Isr()  → 同步时钟中断 → 同上
```

关键点:
- 中断只做 **标记**，不做重活
- 实际数据处理在 while(1) 的 ① 协议栈轮询 中完成
- 中断中访问 SPI 时不关全局中断 (ISR 版本), while(1) 中访问 SPI 时关全局中断 (防冲突)

## 四、GPIO 操作 — 查表 + 位带

```
Application.c 定义两张只读表:

s_DO_Pins[16] = { {GPIOD,0}, {GPIOD,1}, {GPIOD,8}, {GPIOD,9},
                  {GPIOD,10}, {GPIOD,14}, {GPIOD,15},
                  {GPIOE,7}, {GPIOE,8}, {GPIOE,9}, {GPIOE,10},
                  {GPIOE,11}, {GPIOE,12}, {GPIOE,13}, {GPIOE,14}, {GPIOE,15} }

s_DI_Pins[8]  = { {GPIOG,15}, {GPIOB,3}, {GPIOB,4}, {GPIOB,5},
                  {GPIOB,6}, {GPIOB,7}, {GPIOB,8}, {GPIOB,9} }
```

读写都是 O(1), 单条汇编指令:

```
DO_Write(ch, val):
  p = &s_DO_Pins[ch]
  *(BITBAND(p->port_addr + 0x14, p->pin)) = val   // ODR 位带别名区

DI_Read(ch):
  p = &s_DI_Pins[ch]
  return *(BITBAND(p->port_addr + 0x10, p->pin))   // IDR 位带别名区
```

BITBAND 原理: 外设区 0x40000000 映射到 0x42000000，每个 bit 展开为 32-bit word，LDR/STR 单指令读写一个位。

## 五、Flash 持久化参数

```
物理地址: 0x080E0000 (Sector 11, 128KB)

RAM 中对应结构体:
  typedef struct {
      float    sonar_zero_offset;     // 4 bytes
      float    sonar_scale_factor;    // 4 bytes
      uint16_t modbus_slave_addr;     // 2 bytes
      uint16_t baudrate_code;         // 2 bytes
      uint32_t reserved[8];          // 32 bytes 扩展预留
      uint32_t checksum;             // 4 bytes (XOR 自动)
  } PersistentParams_t;              // 总共 48 bytes

全局实例: g_PersistentParams @ ecat_api.c (RAM 中)
```

### 上电恢复 (自动)

```
App_Load_Params_From_Flash()
  1. Flash 全是 0xFF → 新芯片, 跳过, RAM 保持全0默认值
  2. Flash 有数据 → memcpy 到 g_PersistentParams
  3. 校验 checksum (XOR 所有 uint32_t)
     ├─ 通过 → 参数恢复成功, 业务代码直接用
     └─ 失败 → memset(全0), 安全默认值
```

### 保存 (业务代码触发)

```
业务代码:
  g_PersistentParams.sonar_zero_offset = 1.5f;
  ECAT_SetParamDirty();   // g_bConfigDirty = 1

下一次 while(1) 循环:
  ECAT_Stack_MainLoop() 中的 ③:
    g_bConfigDirty == 1 → App_Save_Params_To_Flash()
      1. ComputeChecksum() 算 XOR
      2. memcmp 对比 Flash 已有内容
         └─ 相同 → 跳过, 不擦写 (减少磨损)
      3. HAL_FLASH_Unlock()
      4. Erase Sector 11
      5. HAL_FLASH_Program 逐 32-bit 字写入
      6. HAL_FLASH_Lock()
      7. g_bConfigDirty = 0
```

### 新增参数只需一步

在 `ecat_api.h` 的 `PersistentParams_t` 中加字段，例如:

```c
uint16_t new_param;  // 加在 reserved 前面
```

保存/恢复/校验 全部自动适配，App_Flash.c 不用改。

| 变量 | 住址 | 谁写 | 谁读 |
|------|------|------|------|
| `g_PersistentParams` | `ecat_api.c` (RAM) + Flash | 业务代码 | 业务代码 |

## 七、文件职责一览

```
main.c          入口, 初始化顺序, while(1)
Application.c   你的业务代码: 消抖 → 读写 GPIO → 映射到 PDO
Application.h   GPIO 查表结构 + 回调声明
ecat_api.c      桥接: 函数指针绑定, 看门狗, Flash 自动保存触发
ecat_api.h      统一 API: ECAT_Set/Get, PersistentParams_t, 回调注册
App_Flash.c     Flash 物理操作: 擦除/写入/校验/恢复
App_Flash.h     Flash 接口: g_bConfigDirty, Load/Save
TR8253Evb.c     平台移植: SPI 收发, 中断回调, 定时器
sys.h           位带宏 BITBAND + 端口位宏 PAout/PBin/...
```

## 八、业务工程师只需关心的接口

```c
// === main.c 初始化次序 ===
ECAT_RegisterPeriodicTask(App_Periodic_Task);   // 注册你的周期回调
ECAT_RegisterSafeOutput(App_Safe_Output_State);  // 注册安全输出回调
App_Hardware_Init();                              // 初始化你的外设
ECAT_Stack_Init();                                // 启动协议栈

// === Application.c 你的业务 ===
void App_Periodic_Task(void) {
    // 读输入, 写输出, 读传感器, 更新 PDO
}
void App_Safe_Output_State(void) {
    // 通信断了, 把所有输出拉到安全状态
}

// === GPIO ===
DO_Write(ch, val);      // 单路输出
DI_Read(ch);             // 单路输入
DO_WriteAll(val);        // 全部输出
DI_ReadAll();            // 全部输入

// === Flash 参数 ===
g_PersistentParams.你的字段 = 新值;
ECAT_SetParamDirty();   // 标记需要保存

// === 类型安全的 PDO 读写 (数值型字段) ===
ECAT_Set(对象字段, 值);
ECAT_Get(对象字段, 目标变量);
```
